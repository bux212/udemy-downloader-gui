# Локальное описание проекта и правок

Этот файл — справка **только для локальной машины**: что за проект и какие изменения вносились в коде. **Здесь нет инструкций по отправке в Git** (push, remote и т.п.).

## 1. Кратко о проекте

| Пункт | Описание |
|--------|----------|
| Название | **Udeler** (Udemy Downloader GUI) |
| Назначение | Desktop‑приложение (Electron) для скачивания **ваших** курсов Udemy |
| Точка входа | `main.js` → окно загружает `app/index.html` |
| Основная логика UI и загрузок | `app/app.js` |
| Интеграция с API Udemy | `app/core/services/udemy.service.js` |
| Настройки | `app/helpers/settings.js` |
| Сборка установщика | `npm run build` (electron-builder, артефакты в `dist/`) |

Запуск из исходников: `npm install`, затем `npm start` или `npm run dev`.

## 2. Что меняли (только локальные правки в этой сессии)

**Изменённый файл:** `app/app.js` (функция `getPlaylist` и ветка скачивания HLS `application/x-mpegurl`).

### Проблема

- В `.m3u8` Udemy часто даёт **относительные** URL (`video/.../media.m3u8`, сегменты без `https://`).
- Старый код делал `fetch()` по строке как есть → Electron относил путь к странице `file://.../app/index.html` → запросы вида `file:///D:/.../app/video/...` и ошибки `ERR_FILE_NOT_FOUND`, `getFile_Error: Failed to fetch`.
- Часть курсов отдаёт HLS **не через `.ts`**, а через **fMP4** (`.m4s`) и `#EXT-X-MAP`; старый код искал только `.ts` → список сегментов был пустой, лекция «завершалась» без видео, интерфейс мог показывать успех.

### Что сделано

1. **Резолвинг URL** относительно URL плейлиста: `new URL(line, url).toString()` для вариантов и сегментов.
2. **Master playlist** (`#EXT-X-STREAM-INF`): выбор варианта с максимальным разрешением (по `RESOLUTION`), затем рекурсивный вызов `getPlaylist` для выбранного `.m3u8`.
3. **Media playlist**: сбор всех не‑комментарийных URI‑строк (в т.ч. `.m4s`, не только `.ts`); поддержка **`#EXT-X-MAP:URI="..."`** (init‑сегмент).
4. Если после разбора **не осталось ни одного сегмента** (`list.length === 0`): лог `HLS_EMPTY_PLAYLIST`, сброс курса в состояние ошибки (`download-error`), а не «успешное» завершение без файлов.
